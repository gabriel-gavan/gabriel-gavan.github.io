Playwright Locators: getByRole, getByLabel, getByText

Semantic locators describe elements by their purpose (role, name, label), mirroring how users experience the UI. They are more durable than CSS/XPath because they track accessible names, not DOM structure.

getByRole (preferred for buttons, links, headings)

// Click the primary submit button by its accessible name
await page.getByRole('button', { name: 'Submit' }).click();

// Find a link by its text label
await page.getByRole('link', { name: 'Pricing' }).click();

// Narrow by level for headings
await expect(page.getByRole('heading', { level: 1 })).toBeVisible();

getByLabel / getByPlaceholder (for form controls)

// Use visible label
await page.getByLabel('Email').fill('user@example.com');
await page.getByLabel('Password').fill('secret');

// Fallback if label is missing
await page.getByPlaceholder('Search products').fill('Laptop');

getByText and getByTestId

// Assert a welcome message
await expect(page.getByText('Welcome back')).toBeVisible();

// For custom widgets lacking semantics, add data-testid
await page.getByTestId('cart-icon').click();

Chaining and filtering

// Limit search to a specific card
const card = page.getByRole('article').filter({ hasText: 'Pro Plan' });
await card.getByRole('button', { name: 'Choose' }).click();

// Locate a row containing a specific cell
const row = page.getByRole('row').filter({ has: page.getByRole('cell', { name: /Active/ }) });
await expect(row).toHaveCount(1);

Anti‑patterns to avoid

  • Brittle CSS like #root > div:nth-child(2) > button; prefer roles/labels.
  • Hardcoding text when a better role+name exists (e.g., button name).
  • Using textContent() polling instead of expect(locator).toHaveText(...).

Exercises

  1. Sign in using getByLabel for inputs and getByRole for submit. Assert URL is /dashboard.
  2. Replace two CSS selectors in your suite with role/label equivalents. Document why the new ones are more stable.
  3. Locate a table row containing a user name with chaining + has/hasText.
Tip: If you must use custom selectors, prefer data-testid attributes that are stable and reviewable over brittle DOM paths.
