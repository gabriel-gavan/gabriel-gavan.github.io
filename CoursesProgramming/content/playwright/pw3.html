Running and Debugging Tests

Efficient debugging saves hours. Learn the CLI filters, headed mode, inspector, and trace viewer to understand failures quickly.

CLI basics

# Run all
npx playwright test
# Filter by title or file
npx playwright test login           # spec name contains "login"
npx playwright test tests/login.spec.ts
# By project (browser)
npx playwright test --project=chromium
# Headed mode and slowmo for visibility
npx playwright test --headed --slow-mo=300
# Reporters
npx playwright test --reporter=line,html
# Only run tests matching a grep
npx playwright test --grep "@smoke"

Debugging tools

# Start inspector with UI controls
npx playwright test --debug

# Pause at a specific line
await page.pause();

# See console logs and network in the terminal
DEBUG=pw:api npx playwright test

Open Trace Viewer

Traces show a timeline with DOM snapshots, network, console, and source.

# After a run with tracing enabled
npx playwright show-trace test-results/<run-id>/trace.zip

Record videos and screenshots

// playwright.config.ts
export default defineConfig({
  use: { video: 'retain-on-failure', screenshot: 'only-on-failure', trace: 'on-first-retry' }
});

Useful patterns

  • Add @smoke, @regression, @wip tags in test titles and filter with --grep.
  • Prefer expect(...) assertions to wait for states, instead of manual waitForTimeout.
  • Use --project to repro browser‑specific issues locally.

Practice

  1. Make a test fail, run with --debug, step through actions, and inspect locators.
  2. Enable trace: 'on-first-retry', reproduce a failure, and open the trace to find the cause.
  3. Tag two tests as @smoke and run only those.
Tip: Check the Console tab in Trace Viewer for unexpected errors before focusing on selectors.
