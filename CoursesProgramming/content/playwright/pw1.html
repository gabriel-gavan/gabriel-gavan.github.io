Playwright: What It Is and Why It Matters

Playwright is a modern end‑to‑end testing framework for web apps. It automates Chromium, Firefox, and WebKit with a single API, emphasizing reliability through auto‑waiting, resilient locators, and rich debugging tools. It is designed for CI from day one and works well for UI, API, and component testing.

Why teams pick Playwright

  • Single API for all major browsers (Chromium, Firefox, WebKit), including mobile emulation.
  • Auto‑waiting for elements to be actionable; fewer sleep calls and race conditions.
  • Powerful Locator engine (roles, labels, text, test IDs) and chainable filters.
  • First‑class debugging: Inspector, Trace Viewer, videos, console/network/event logs.
  • Integrated test runner with parallelism, sharding, retries, and fixtures.

Install and scaffold

# Initialize a Node project
npm init -y
# Install Playwright Test runner
npm i -D @playwright/test
# Download browser binaries
npx playwright install
# Optional: scaffold example tests and config
npx playwright test --init

Project setup (TS and JS)

# TypeScript (recommended)
npm i -D typescript ts-node @types/node

# Use .mjs files or set "type": "module" in package.json

Your first UI test

import { test, expect } from '@playwright/test';

test('homepage shows headline', async ({ page }) => {
  await page.goto('https://example.com');
  await expect(page).toHaveTitle(/Example/);
  await expect(page.getByRole('heading', { level: 1 })).toBeVisible();
});

Codegen (record selectors)

# Launch the recorder; click around to generate code
npx playwright codegen https://example.com

# Generate into a test file
echo "import { test, expect } from '@playwright/test';\n" > tests/example.spec.ts
npx playwright codegen https://example.com --output tests/example.spec.ts

Recommended folder layout

project/
  playwright.config.ts
  tests/
    smoke.spec.ts
  pages/
    HomePage.ts
  fixtures/
    auth.ts
  test-data/
    users.json

Run tests

# All tests
npx playwright test
# Headed debugging
npx playwright test --headed --project=chromium
# Open last HTML report
npx playwright show-report

Common pitfalls to avoid

  • Avoid fixed sleeps; rely on auto‑waiting and expect(...) with timeouts.
  • Prefer semantic locators (getByRole, getByLabel, getByText) over brittle CSS/XPath.
  • Keep tests independent; do not rely on execution order or shared state.

Exercises

  1. Open a site you use daily and assert title, URL, and a visible H1.
  2. Use getByRole to click the main call‑to‑action. Assert a success message appears.
  3. Record a short flow with codegen, then refactor selectors to semantic locators.
  4. Capture a screenshot after the click and store it in test-results/.
Tip: Keep one test = one user goal (login, add to cart). Short, intention‑revealing tests are more reliable and easier to debug.
