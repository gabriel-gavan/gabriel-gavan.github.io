Project Structure and Configuration

A predictable structure keeps tests scalable and maintainable. Store tests, page objects, fixtures, and config in consistent places so teams can navigate quickly.

Recommended layout

project/
  playwright.config.ts
  tests/
    smoke.spec.ts
    login.spec.ts
    checkout.spec.ts
  pages/
    LoginPage.ts
    CartPage.ts
    components/
      Modal.ts
  fixtures/

  helpers/

Multi‑environment config

Switch base URLs and flags by environment variables or separate files:

# package.json scripts
{
  "scripts": {
    "test:dev":   "cross-env BASE_URL=http://localhost:3000 npx playwright test",
    "test:stg":   "cross-env BASE_URL=https://staging.example.com npx playwright test",
    "test:mobile": "npx playwright test --project='Mobile Chrome'"
  }
}
// playwright.config.ts (read env)
const baseURL = process.env.BASE_URL || 'http://localhost:3000';
export default defineConfig({ use: { baseURL } });

Global setup and authenticated state

Create a user session once and reuse it via storageState to speed up suites.

// global-setup.ts
import { chromium, FullConfig } from '@playwright/test';

async function globalSetup(config: FullConfig) {
  const browser = await chromium.launch();
  const context = await browser.newContext();
  const page = await context.newPage();
  await page.goto(process.env.BASE_URL || 'http://localhost:3000');
  // Perform login flow
  await page.getByLabel('Email').fill('user@example.com');
  await page.getByLabel('Password').fill('secret');
  await page.getByRole('button', { name: 'Log in' }).click();
  await page.waitForURL('**/dashboard');
  await context.storageState({ path: 'state.json' });
  await browser.close();
}
export default globalSetup;
// playwright.config.ts
export default defineConfig({
  globalSetup: require.resolve('./global-setup'),
  use: { storageState: 'state.json' }
});

Naming and organization tips

  • Group related tests by feature (e.g., tests/checkout/).
  • Keep page objects lean: expose intention‑revealing methods; avoid business assertions there.
  • Put one major flow per test file to simplify local runs and triage.

Practice

  1. Extract a LoginPage class with login(email, pass) and reuse it in tests.
  2. Implement a global setup to persist storageState for authenticated tests.
  3. Add a stg script that points to a staging base URL; verify process.env.BASE_URL is used.
Tip: Keep config deterministic in CI: set viewport, timezone, locale, and mock non‑deterministic data when visual or timing‑sensitive.
