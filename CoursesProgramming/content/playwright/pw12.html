Trace Viewer and Screenshots

Capture traces, screenshots, and videos to investigate failures quickly. Traces include DOM snapshots, network, console logs, and action timings.

Enable tracing, screenshots, videos

// playwright.config.ts
export default defineConfig({
  use: {
    trace: 'on-first-retry',       // or 'retain-on-failure' | 'on'
    screenshot: 'only-on-failure', // or 'on'
    video: 'retain-on-failure',
  }
});

View a trace

# After a run with tracing enabled
npx playwright show-trace test-results/<run-id>/trace.zip

Stable screenshots

  • Disable animations/transitions during capture (set prefers-reduced-motion: reduce or CSS overrides).
  • Fix viewport size and device scale factor; ensure deterministic fonts in CI.
  • Mask dynamic regions (timestamps, random avatars) when using toHaveScreenshot.

Examples

// Page-level snapshot
await expect(page).toHaveScreenshot('dashboard.png');

// Element-level snapshot with masks
await expect(page.locator('#chart')).toHaveScreenshot('chart.png', {
  mask: [page.locator('.ticker'), page.locator('.dynamic-ad')],
});

Exercises

  1. Enable video capture and compare a failing vs. passing run.
  2. Capture a masked screenshot for a widget with dynamic timestamps.
  3. Open a trace and identify the first failing assertion and its preceding action.
Tip: Traces are most useful when tests are short and assertions are placed right after the action they validate.
