Visual Snapshot Testing

Catch unintended UI changes by comparing screenshots across commits. Keep visuals deterministic to avoid noisy diffs.

Usage

// Page-level comparisons
await expect(page).toHaveScreenshot('home.png');

// Element-level comparisons with options
await expect(page.locator('#hero')).toHaveScreenshot('hero.png', {
  maxDiffPixelRatio: 0.01,
  animations: 'disabled',
  caret: 'hide',
});

Stability tips

  • Set a fixed viewport and device scale factor; pin fonts in CI.
  • Mock timestamps, random IDs, and network data that can change per run.
  • Disable animations/transitions or use prefers‑reduced‑motion in tests.
  • Mask dynamic areas (ads, tickers) to prevent false diffs.

Helpers

export async function stableScreenshot(page, locator, name){
  await page.addStyleTag({ content: '* { transition: none !important; animation: none !important; }' });
  await expect(locator).toHaveScreenshot(name, { caret: 'hide', animations: 'disabled' });
}

Exercises

  1. Baseline a component screenshot and verify a CSS change is detected.
  2. Write a helper to mask dynamic areas (e.g., ads) and use it on a news card.
  3. Stabilize fonts in CI and verify diffs disappear for unchanged UI.
Tip: Visual tests should be selective: target critical, styled areas rather than full pages when possible to reduce noise and review burden.
