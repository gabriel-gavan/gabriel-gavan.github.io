DOM Events and Patterns

Events notify your code when the user interacts. Organize handlers for performance and maintainability.

Event basics

document.querySelector('#btn').addEventListener('click', (e) => {
  e.preventDefault();
  console.log('Clicked');
}, { once: false, passive: false, capture: false });

Delegation

const list = document.querySelector('#todo-list');
list.addEventListener('click', (e) => {
  const item = e.target.closest('li');
  if (!item || !list.contains(item)) return;
  item.classList.toggle('done');
});

Throttle and debounce

function throttle(fn, wait){
  let last = 0;
  return (...args) => {
    const now = Date.now();
    if(now - last >= wait){ last = now; fn(...args); }
  };
}

function debounce(fn, wait){
  let t;
  return (...args) => {
    clearTimeout(t);
    t = setTimeout(() => fn(...args), wait);
  };
}

window.addEventListener('scroll', throttle(() => {
  // expensive work here
}, 100), { passive: true });

Abort and teardown

const ctrl = new AbortController();
const { signal } = ctrl;

window.addEventListener('resize', debounce(() => {/*...*/}, 200), { signal });
// Later
ctrl.abort(); // removes listeners registered with this signal

Exercises

  1. Implement a todo list with add/remove and a delegated click to toggle completion.
  2. Throttle a scroll handler to run at most once every 100 ms.
  3. Use AbortController to remove listeners on teardown.
  4. Mark wheel/scroll listeners as passive unless you call preventDefault.
Tip: Prefer delegation for dynamic lists to avoid attaching thousands of listeners. Use passive listeners to improve scroll performance.
