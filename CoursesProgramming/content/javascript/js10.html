ES Modules: import/export

Split code into modules with clear interfaces. Prefer ESM in modern browsers and Node (with "type": "module").

Named vs default exports

// math.js
export function add(a,b){ return a+b; }
export const PI = 3.14159;
export default function sub(a,b){ return a-b; }

// main.js
import sub, { add, PI } from './math.js';
console.log(add(2,2), sub(4,1), PI);

Re-exports and index files

// utils/index.js
export * from './dates.js';
export * as http from './http.js';

Dynamic import and top-level await

// Lazy-load a heavy module
const { heavyFn } = await import('./heavy.js');
const result = await heavyFn();

Browser and Node setup

// index.html
// <script type="module" src="./main.js"></script>

// package.json (Node)
{
  "type": "module",
  "scripts": { "start": "node main.js" }
}

Common pitfalls

  • Use file extensions in browser imports (./module.js).
  • Don’t mix require() and import in the same file; convert to ESM fully.
  • For Node, configure type: module or use .mjs.

Exercises

  1. Refactor a script into ESM modules with named and default exports.
  2. Set type="module" in HTML and test in the browser.
  3. Add a dynamic import for a rarely used feature.
Tip: Keep modules small and cohesive; group re-exports in an index.js to simplify imports.
