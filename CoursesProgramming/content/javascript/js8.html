Async JavaScript: Callbacks, Promises, async/await

Async code prevents UI jank during I/O. Promises model eventual completion; async/await makes chains readable.

From callbacks to promises to async/await

// Promise style
fetch('/api/user')
  .then(r => r.json())
  .then(u => console.log(u))
  .catch(err => console.error(err));

// async/await style
try {
  const r = await fetch('/api/user');
  if(!r.ok) throw new Error('Bad response');
  const u = await r.json();
  console.log(u);
} catch (e) {
  console.error(e);
}

Parallel vs sequential

// Sequential (slower)
const a = await fetch('/a').then(r => r.json());
const b = await fetch('/b').then(r => r.json());

// Parallel (faster)
const [a2, b2] = await Promise.all([
  fetch('/a').then(r => r.json()),
  fetch('/b').then(r => r.json()),
]);

Promise utilities

await Promise.race([p1, p2]);      // first to settle
await Promise.any([p1, p2]);       // first to fulfill (ignores rejections)
await Promise.allSettled([p1, p2]); // get outcomes without throwing

Error handling and cancellation

  • Always check response.ok; normalize error shapes.
  • Use AbortController to cancel long requests.
  • Surface friendly UI messages; log details for debugging.

Exercises

  1. Wrap fetch in a helper that times out after 5 seconds.
  2. Retry transient failures up to 3 times with exponential backoff.
  3. Parallelize two independent requests with Promise.all.
  4. Use Promise.allSettled to show partial results even if some fail.
Tip: Avoid await inside Array.map without collecting promises. Build an array of promises and await Promise.all.
