Variables: var, let, and const

let and const are block‑scoped; var is function‑scoped and hoisted differently. Prefer const by default and let when reassignment is intentional.

Scoping and hoisting

// var is function-scoped and hoisted with undefined initialization
console.log(a); // undefined
var a = 1;

// let/const are block-scoped and in the Temporal Dead Zone until declared
// console.log(b); // ReferenceError
let b = 2;

if (true) {
  let x = 1; // not visible outside
}
// console.log(x); // ReferenceError

const binds the reference, not the value

const user = { name: 'Ava' };
user.name = 'Mia'; // OK (mutation)
// user = {} // Error (reassignment)

Object.freeze(user); // Shallow freeze; nested objects still mutable

Primitive vs reference

  • Primitives (number, string, boolean, null, undefined, symbol, bigint) copy by value.
  • Objects/arrays/functions copy references; mutations affect all references.

Strict mode and globals

'use strict';
// Accidental globals throw instead of silently creating window.foo
// foo = 1; // ReferenceError

Best practices

  • Default to const; use let for variables that change.
  • Avoid var to prevent hoisting and scoping pitfalls.
  • Initialize variables close to first use; limit scope width.
