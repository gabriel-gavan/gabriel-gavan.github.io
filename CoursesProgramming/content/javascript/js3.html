Functions and Arrow Functions

Functions encapsulate reusable logic. Arrow functions offer a concise syntax and lexical this binding, which differs from traditional functions.

Declaration vs. expression

// Declaration (hoisted)
function add(a, b) { return a + b; }

// Expression (not hoisted the same way)
const mul = function(a, b) { return a * b; };

// Arrow (lexical this)
const pow = (a, b) => a ** b;

Parameters and defaults

function greet(name = 'friend', { excited = false } = {}){
  const msg = `Hello, ${name}${excited ? '!' : ''}`;
  return msg;
}

function sum(...nums){ return nums.reduce((a,b) => a+b, 0); }

Arrow gotchas

  • No own this, arguments, super, or new.target.
  • Not constructible (new with arrow throws).
  • Implicit return for single expressions; wrap object literals in (...).
const makeUser = (name) => ({ name }); // parentheses to return object

Closures and currying

function makeCounter(){
  let n = 0;
  return () => ++n; // closure over n
}

const addN = n => x => x + n; // curried add

Best practices

  • Keep functions small; do one thing well.
  • Prefer pure functions for predictable outcomes.
  • Use arrow functions for callbacks; regular functions for methods needing dynamic this.
