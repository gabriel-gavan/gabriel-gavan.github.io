Testing with Jest (Intro)

Write unit tests for pure functions to get fast feedback. Aim for small, isolated tests with clear Arrange‑Act‑Assert steps.

Basics

describe('math', () => {
  test('adds', () => { expect(2 + 2).toBe(4); });
  test('throws', () => {
    const willThrow = () => { throw new Error('boom'); };
    expect(willThrow).toThrow('boom');
  });
});

Async tests

test('loads data', async () => {
  const api = { get: jest.fn().mockResolvedValue({ id: 1 }) };
  const data = await api.get('/users/1');
  expect(api.get).toHaveBeenCalledWith('/users/1');
  expect(data).toEqual({ id: 1 });
});

Coverage

# package.json
{ "scripts": { "test:cov": "jest --coverage" } }

Exercises

  1. Test edge cases and error conditions.
  2. Measure coverage and identify gaps.
  3. Mock a module and assert how it was called.
Tip: Prefer testing public behavior over internal implementation details. Tests should enable refactoring, not block it.
