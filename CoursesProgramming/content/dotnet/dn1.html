.NET and C#: A Quick Tour

.NET is a cross‑platform runtime and base class library. C# is the flagship language, featuring strong typing, LINQ, async/await, pattern matching, and records.

Setup

# Windows
winget install Microsoft.DotNet.SDK.8

# macOS (Homebrew)
brew install --cask dotnet-sdk

# Linux (Debian/Ubuntu)
wget https://dot.net/v1/dotnet-install.sh -O dotnet-install.sh
bash dotnet-install.sh --channel 8.0

# Verify
dotnet --info

Create your first app

dotnet new console -n HelloDotnet
cd HelloDotnet

// Program.cs (top-level statements)
Console.WriteLine("Hello, .NET!");

dotnet run

Project layout and CLI

// Create solution and projects
dotnet new sln -n MyApp
cd MyApp
dotnet new console -n App
 dotnet sln add App/App.csproj

// Build / Run / Test
dotnet build
cd App && dotnet run
# Add tests
dotnet new xunit -n App.Tests
 dotnet sln add App.Tests/App.Tests.csproj
 dotnet test

NuGet packages

# Add a package to a project
dotnet add App/App.csproj package Newtonsoft.Json --version 13.*

# Restore and build
dotnet restore && dotnet build

Publish

# Framework-dependent (portable)
dotnet publish -c Release -o out

# Self-contained single-file (example: win-x64)
dotnet publish -c Release -r win-x64 /p:PublishSingleFile=true /p:PublishTrimmed=true

Exercises

  1. Create a console app that reads a name and prints a greeting.
  2. Add a method to compute the average of a list of numbers. Unit test it with xUnit.
  3. Add a NuGet dependency and use it from Program.cs (e.g., JSON serialization).
Tip: Keep SDKs pinned with a global.json for reproducible builds in CI.
