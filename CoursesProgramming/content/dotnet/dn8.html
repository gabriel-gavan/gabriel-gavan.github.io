LINQ Basics

Query collections fluently with LINQ. Operations are deferred until you enumerate or call a terminal method.

Method syntax

var nums = new[]{1,2,3,4,5,6};
var query = nums.Where(n => n % 2 == 0).Select(n => n * n);
var squares = query.ToArray(); // execution happens here

Group, join, distinct

var byCity = users.GroupBy(u => u.City).Select(g => new { City = g.Key, Avg = g.Average(u => u.Points) });
var joined = orders.Join(customers, o => o.CustomerId, c => c.Id, (o, c) => new { o.Id, c.Name });
var distinctNames = users.Select(u => u.Name).Distinct().ToList();

Order and paging

var page = users.OrderBy(u => u.Name).Skip(20).Take(10).ToList();

Gotchas

  • Multiple enumeration: materialize once with ToList() if you iterate repeatedly.
  • Side effects in Select/Where lead to confusion; keep operators pure.

Exercises

  1. Group users by city and compute average points per group.
  2. Project items into DTOs using Select.
  3. Implement paging with Skip/Take and verify counts.
Tip: When composing complex queries, extract parts into named methods to keep pipelines readable.
