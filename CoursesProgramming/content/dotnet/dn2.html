Types, Variables, and Operators

C# is strongly typed. Prefer explicit types for clarity or var when the type is obvious on the right‑hand side.

Value vs reference types

int count = 5;              // value type
bool isActive = true;       // value type
string name = "Mia";         // reference type
DateTime when = DateTime.UtcNow; // struct (value type)
  • Value types (structs) live on the stack, copied by value.
  • Reference types (classes, arrays, strings) live on the heap; variables hold references.

Nullable reference types (NRT)

#nullable enable
string? nickname = null; // must check for null before deref
string title = "Engineer"; // non-nullable

Parsing and formatting

if (int.TryParse("42", out var n)) { /* use n */ }

using System.Globalization;
var price = 19.99m;
var us = new CultureInfo("en-US");
Console.WriteLine(price.ToString("C", us)); // $19.99

Operators

  • Arithmetic: + - * / %
  • Comparison: == != < > <= >=
  • Logical: && || !
  • Null-coalescing: ??, null‑propagating: ?.

Best practices

  • Enable nullable reference types in your project for safer APIs.
  • Prefer decimal for money; avoid double rounding errors.
  • Use TryParse overloads to avoid exceptions on bad input.
