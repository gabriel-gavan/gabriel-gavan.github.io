Async/await and Tasks

Asynchronous code in C# is built on Task and the async/await keywords. They help you orchestrate I/O without blocking threads.

HttpClient reuse and cancellation

static readonly HttpClient http = new HttpClient();

static async Task LoadAsync(string url, CancellationToken ct = default)
{
  using var resp = await http.GetAsync(url, ct);
  resp.EnsureSuccessStatusCode();
  return await resp.Content.ReadAsStringAsync(ct);
}

Parallel requests

var a = LoadAsync("https://a.example");
var b = LoadAsync("https://b.example");
var results = await Task.WhenAll(a, b);

Timeouts

using var cts = new CancellationTokenSource(TimeSpan.FromSeconds(5));
var text = await LoadAsync("https://example.com", cts.Token);

IAsyncEnumerable

static async IAsyncEnumerable StreamAsync()
{
  for (int i=0; i<3; i++) { await Task.Delay(100); yield return i; }
}
await foreach (var n in StreamAsync()) Console.WriteLine(n);

Exercises

  1. Run two HTTP calls in parallel and await both with Task.WhenAll.
  2. Add a cancellation token to stop work after a timeout.
  3. Stream a sequence with IAsyncEnumerable and verify ordering.
Avoid: Blocking on async (e.g., .Result) can deadlock UI contexts. In libraries, consider ConfigureAwait(false).
