Encapsulation and Access Modifiers

Protect invariants by keeping fields private and exposing behavior through public methods.

Access levels

  • public: everywhere
  • protected: same package + subclasses
  • (package‑private): same package (no modifier)
  • private: same class only

Example

class Account {
  private double balance;
  public void deposit(double amount){ if(amount > 0) balance += amount; }
  public void withdraw(double amount){
    if(amount <= 0) throw new IllegalArgumentException("amount");
    if(amount > balance) throw new IllegalStateException("insufficient funds");
    balance -= amount;
  }
  public double getBalance(){ return balance; }
}

Defensive copies

class Order {
  private final java.util.Date createdAt;
  Order(java.util.Date createdAt){ this.createdAt = new java.util.Date(createdAt.getTime()); }
  public java.util.Date getCreatedAt(){ return new java.util.Date(createdAt.getTime()); }
}

Exercises

Tip: Prefer immutability for value objects and expose behavior (not setters) for entities.
