Exceptions and try/catch

Handle exceptional cases explicitly and keep normal flows readable.

Checked vs unchecked

  • Checked: must be declared or handled (e.g., IOException).
  • Unchecked: subclasses of RuntimeException; often programming errors.

Basic try/catch

try {
  Integer.parseInt("abc");
} catch (NumberFormatException e) {
  System.out.println("Bad number");
}

try-with-resources

import java.nio.file.*;
try (var lines = Files.lines(Path.of("data.txt"))) {
  long count = lines.count();
} catch (Exception e) {
  // handle
}

Custom exceptions

class DomainException extends RuntimeException {
  DomainException(String msg){ super(msg); }
}

void withdraw(int amount){
  if(amount < 0) throw new DomainException("amount must be positive");
}

Exercises

  1. Create a custom exception for domain errors.
  2. Use try-with-resources to auto-close files.
  3. Normalize exception messages and include context.
Tip: Prefer informative messages and include key values. Avoid swallowing exceptions; rethrow with cause when needed.
