Java Collections: Lists and Maps

The Collections Framework provides ready‑made data structures. Understanding their trade‑offs helps you write efficient code.

Lists

import java.util.*;

List items = new ArrayList<>();
items.add("alpha");
items.add("beta");
System.out.println(items.get(0)); // alpha

List immutable = List.of("a", "b"); // unmodifiable
  • ArrayList: fast random access, append‑heavy workloads.
  • LinkedList: efficient insertions/removals in the middle (rarely needed).

Maps

Map scores = new HashMap<>();
scores.put("ava", 10);
scores.put("rui", 7);
System.out.println(scores.get("ava")); // 10

Map small = Map.of("a",1, "b",2); // unmodifiable
  • HashMap: average O(1) get/put.
  • TreeMap: sorted by key (O(log n)).
  • LinkedHashMap: predictable iteration order (insertion access options).

Iteration patterns

scores.forEach((k,v) -> System.out.println(k + ":" + v));
for (var e : scores.entrySet()) { /* ... */ }

scores.merge("ava", 1, Integer::sum); // counting
scores.computeIfAbsent("new", k -> 0);

Exercises

  1. Create a Map<String, List<String>> that groups users by city.
  2. Count occurrences of words in a sentence using Map.merge.
  3. Iterate with forEach((k,v) -> ...) to print a leaderboard.
Tip: Favor interfaces (List, Map) over implementations (ArrayList, HashMap). Prefer unmodifiable views for read‑only data.
