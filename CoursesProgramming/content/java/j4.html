Java OOP: Classes and Objects

Classes define the shape of your data and the operations that act on it. Objects are instances of classes at runtime.

Defining a class

package app.model;

import java.util.Objects;

public class User {
  private final String name;
  private int points = 0;

  public User(String name) {
    this.name = Objects.requireNonNull(name, "name");
  }
  public String getName() { return name; }
  public int getPoints() { return points; }
  public void addPoints(int delta) { if (delta > 0) points += delta; }

  @Override public String toString(){ return name + "(" + points + ")"; }
  @Override public boolean equals(Object o){
    if (this == o) return true;
    if (!(o instanceof User u)) return false;
    return name.equals(u.name);
  }
  @Override public int hashCode(){ return name.hashCode(); }
}

Using the class

public class Main {
  public static void main(String[] args) {
    User u = new User("Ava");
    u.addPoints(5);
    System.out.println(u);
  }
}

Records (Java 16+)

public record Point(int x, int y) { }
var p = new Point(1, 2); // immutable data carrier

Design notes

  • Prefer immutability for identity (e.g., name), mutability for counters.
  • Validate inputs in constructors and methods.
  • Override equals/hashCode for value semantics (map keys, set membership).
