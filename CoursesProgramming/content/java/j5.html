Methods and Overloading

Use overloading to offer convenient APIs with the same method name but different parameter lists. Don’t confuse overloading (compile‑time) with overriding (runtime dispatch).

Overloading

class MathOps {
  int add(int a, int b){ return a + b; }
  double add(double a, double b){ return a + b; }
  int add(int a, int b, int c){ return a + b + c; }
}

Varargs and defaults

class Strings {
  static String join(String sep, String... parts){
    return String.join(sep, parts);
  }
}

Strings.join(", ", "a", "b", "c");

Overriding vs overloading

  • Overloading: same name, different parameters; chosen at compile time.
  • Overriding: subclass provides a new implementation; resolved at runtime via dynamic dispatch.

Optional results

import java.util.Optional;
Optional findScore(String user) {
  // return Optional.empty() if not found
  return Optional.of(42);
}
