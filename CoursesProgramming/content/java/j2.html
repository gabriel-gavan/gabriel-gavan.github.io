Variables, Types, and Operators

Java’s type system helps catch mistakes at compile time. Choose types that match your intent.

Primitives vs references

int age = 21;          // primitive
double price = 19.99;  // primitive
boolean active = true; // primitive
String name = "Ava";    // reference type
  • Primitives live on the stack; references point to heap objects.
  • Use == for primitives; use Objects.equals(a,b) or .equals for references.

Boxing and unboxing

Integer n = 5;        // boxing
int m = n;            // unboxing
List xs = List.of(1,2,3); // collections use wrappers
Note: Avoid unnecessary boxing in hot code; prefer primitives in loops and math.

Numbers and money

// Use BigDecimal for money (avoid double rounding errors)
import java.math.BigDecimal;
BigDecimal price = new BigDecimal("19.99");
BigDecimal qty = BigDecimal.valueOf(3);
BigDecimal total = price.multiply(qty);

Operators

  • Arithmetic: + - * / %
  • Comparison: == != < > <= >=
  • Logical: && || !
  • Assignment: += -= *= /=

Formatting and parsing

import java.text.NumberFormat;
var usd = NumberFormat.getCurrencyInstance(java.util.Locale.US);
System.out.println(usd.format(19.99));
