RAII and Resource Management

Acquire resources in constructors and release them in destructors to ensure cleanup on all paths. RAII underpins exception safety.

Example: file wrapper

struct File {
  FILE* f{};
  explicit File(const char* p){ f = std::fopen(p, "r"); }
  ~File(){ if(f) std::fclose(f); }
  File(const File&) = delete; File& operator=(const File&) = delete; // non-copyable
  File(File&& o) noexcept : f(o.f){ o.f = nullptr; }
  File& operator=(File&& o) noexcept { if(this!=&o){ if(f) std::fclose(f); f=o.f; o.f=nullptr; } return *this; }
};

Exception safety

  • Basic guarantee: no resource leaks; invariants preserved.
  • Strong guarantee: commit/rollback semantics (copy‑and‑swap).
  • No‑throw guarantee: operations marked noexcept.
