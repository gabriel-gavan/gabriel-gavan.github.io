Pointers and Addresses

Manipulate memory addresses directly for powerful low‑level control. Prefer RAII and containers to manual new/delete.

Basics

int v = 42;
int* p = &v;   // address of v
*p = 43;       // changes v
p = nullptr;   // null pointer

Arrays and pointer arithmetic

int a[3] = {1,2,3};
int* q = a;           // points to first element
*(q + 1) = 42;        // writes to a[1]
// Prefer std::vector for dynamic arrays
std::vector vec = {1,2,3}; vec.push_back(4);

Common pitfalls

  • Dangling pointers after delete or leaving scope.
  • Double delete, mismatched new[]/delete[].
  • Leaking memory on early returns/exceptions (use RAII).

Exercises

  1. Allocate an array with new/delete[], then refactor to std::vector.
  2. Identify places where a pointer should be nullptr-checked or replaced by a reference.
  3. Explain how smart pointers eliminate common ownership bugs.
Tip: Prefer containers (std::vector) and smart pointers over raw owning pointers in modern C++.
