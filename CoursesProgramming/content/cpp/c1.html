C++: Compilation and the Toolchain

C++ compiles to native machine code, giving you performance and control. Understanding the build pipeline (preprocess → compile → link) helps diagnose errors quickly.

Minimal program

#include 

int main() {
  std::cout << "Hello, C++";
  return 0;
}

Compile and run (with warnings)

// Linux/macOS (Clang or GCC)
clang++ -std=c++20 -O0 -g -Wall -Wextra -Wpedantic main.cpp -o app && ./app
# or
g++     -std=c++20 -O0 -g -Wall -Wextra -Wpedantic main.cpp -o app && ./app

// Windows (MSVC)
cl /EHsc /std:c++20 /W4 main.cpp && main.exe

Sanitizers (debugging memory/UB)

# Address/UB sanitizers (Clang/GCC)
clang++ -std=c++20 -fsanitize=address,undefined -fno-omit-frame-pointer -g main.cpp -o app && ./app

CMake (portable builds)

# CMakeLists.txt
cmake_minimum_required(VERSION 3.20)
project(hello LANGUAGES CXX)
set(CMAKE_CXX_STANDARD 20)
add_executable(app main.cpp)

# Build
cmake -S . -B build
cmake --build build --config Debug
./build/app

Linking basics

  • Static linking bundles libraries into the binary.
  • Dynamic linking uses shared libraries at runtime.
  • On Windows, prefer MSVC; on Linux/macOS, Clang/GCC are common.

Exercises

  1. Write a function that doubles a number and call it from main.
  2. Build with warnings enabled (-Wall -Wextra or /W4) and fix them.
  3. Enable AddressSanitizer and find a deliberate out‑of‑bounds bug.
  4. Create a minimal CMake project and build it out‑of‑source.
Tip: Treat warnings as errors (-Werror / /WX) in CI to keep code quality high.
